## 目前已经做出来的核心进度

### 1. AgentRuntime 已经成型

现在 `PetWindow` 里已经创建了 `AgentRuntime`，并把 `MemoryStore`、`ReminderStore`、`ToolRegistry` 都接进去了，不是直接把用户消息丢给聊天 API 了。

`AgentRuntime.handle_user_message()` 现在流程是：

```text
用户消息
  ↓
complete_raw 做工具规划
  ↓
解析 tool_calls
  ↓
执行工具 / 生成待确认动作
  ↓
再让模型总结工具结果
  ↓
返回 AgentResult
```

这个已经是标准 Agent Loop 的简化版了。

也就是说，上次说的“先不要急着 MCP，先搭 Agent Runtime”，你已经做出来了。

## 当前最主要的问题

### 问题 1：提醒触发时可能卡 UI

`_check_due_reminders()` 里直接调用了：

```python
result = self.agent_runtime.handle_event(...)
```

这个里面会调模型 API。也就是说提醒到期时，如果网络慢，UI 线程可能直接卡住。

这个建议尽快改。
和普通聊天一样，提醒事件也应该丢给 `ChatWorker` 或单独的 `EventWorker`。

方向：

```text
QTimer 发现提醒到期
  ↓
mark 或 lock 这个 reminder，避免重复触发
  ↓
开 QThread / Worker
  ↓
AgentRuntime.handle_event()
  ↓
回 UI 显示
```

------

### 问题 2：确认动作后的回复没有写入历史

普通聊天 `_handle_reply()` 会：

```python
self.messages.append(...)
self._record_history(...)
```

但是确认动作后的 `_handle_action_reply()` 只显示回复，没有写入 `messages` 和历史。

这会导致一个问题：

```text
用户：打开 xxx 文件夹
Sakura：执行前需要确认
用户点执行
Sakura：已经打开
```

最后这句“已经打开”可能不会进聊天上下文和历史记录。

建议 `_handle_action_reply()` 也记录：

```python
self.messages.append({"role": "assistant", "content": result.reply.text})
self._record_history("assistant", result.reply.text, result.reply.translation)
```

------

### 问题 3：现在还是“JSON Prompt Tool Calling”，不是强约束工具调用

现在 Agent Runtime 是让模型返回 JSON，再自己解析：

```python
first_content = self.api_client.complete_raw(...)
agent_data = _load_json_object(first_content)
tool_calls = _parse_tool_calls(...)
```

这能跑，但会有脆弱性：模型偶尔不按 JSON 输出，或者把工具名/参数写错。

短期可以继续这样。
但后面如果要稳定，建议抽象出两种模式：

```text
PromptJsonPlanner
NativeToolCallPlanner
```

先保留现在这个，后面如果 API 支持原生 tool calling，就可以换 Planner，不影响 UI 和 ToolRegistry。

------

### 问题 4：缺少测试

现在你已经有很多“纯逻辑模块”了：

```text
MemoryStore
ReminderStore
ToolRegistry
NotesStore
TodoStore
AgentRuntime 的 JSON 解析
```

这些非常适合写单元测试。

优先测试：

```text
ReminderStore:
- delay_seconds 能正确生成未来时间
- 过去 trigger_at 会报错
- due_reminders 能筛出到期提醒
- mark_completed 后不再 due

MemoryStore:
- propose 不直接进入 memories
- confirm 后从 pending 移到 memories
- forget 能删除
- category 非法时报错

ToolRegistry:
- requires_confirmation=True 不直接执行
- 未知工具返回失败
```

这一步比现在接 MCP 更重要。

```text

  ├── builtin_tools
```

所以接 MCP 的正确时间点是：


现在如果接 MCP，容易出现：

```text

```

你现在应该先把本地工具闭环跑稳。
MCP 可以作为 0.4，不是 0.2/0.3 的核心。
